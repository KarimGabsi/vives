Maze¶

Inleiding¶

Het doolhofprobleem is een veel gebruikt probleem om zoekalgoritmen op uit te testen. In deze oefening stellen we een doolhof voor als een rooster met n rijen en n kolommen. We hebben in totaal dus n x n vakken. Een vak kan een barrire bevatten, dit zijn vakjes waarop niet kan 'gewandeld' worden.

De uitdaging is om, gegeven een startpositie (links onder in het voorbeeld), een pad naar het doel te vinden (rechts boven in het voorbeeld). De mogelijke richtingen waarin gewandeld kan worden (in die volgorde!) zijn: links, rechts, naar boven en naar onder. Diagonaal kan er niet gewandeld worden. Dit zijn hier onze production rules, die we gaan toepassen op onze huidige toestand of state om te weten welke vakjes we in een volgende stap kunnen nemen. Om die volgende zetten of moves te kunnen bepalen, moeten we naast de production rules de volgende zaken kennen:

  • de positie wat het vakje is waar we staan;
  • de configuratie van het doolhof;
  • het pad dat we al hebben afgelegd.

Het eerste punt is evident, het tweede is nodig omdat we moeten weten waar de barrires en randen zijn, en het laatste punt is nodig omdat we niet op onze stappen willen terugkeren. Als we alle mogelijke moves hebben berekend, dan kiezen we er n uit en gaan zo n vakje verder, waardoor we in een nieuwe toestand komen. De keuze van die move hangt af van het zoekalgoritme dat we gaan toepassen. Daar gaan we in de volgende topics dieper op in gaan. In deze notebook maken we een implementatie van de state-space en de production rules voor het doolhofprobleem. De state-space is de verzameling van alle mogelijke toestanden van het zoekprobleem.

Positie¶

In het doolhof moeten we dus eerst en vooral de verschillende vakjes of posities kunnen aanduiden. Daarvoor definiren we een klasse Position die een (irow, icol)-cordinaat in het doolhof voorstelt. De cordinaat links boven is (0, 0).

In [197]:
class Position:
  
    def __init__(self, irow, icol):
        self.irow = irow
        self.icol = icol

    def __eq__(self, other):
        return self.irow == other.irow and self.icol == other.icol
    
    def __repr__(self):
        return "({}, {})".format(self.irow, self.icol)

Methode __init__ kunnen we zien als de constructor van de klasse (ook al is het in principe geen echte constructor). We willen ook nagaan of twee posities al dan niet gelijk zijn, en daarom overschrijven we de "equality" methode __eq__, die de gelijkheidsoperator == overschrijft. Deze methode retourneert True als de cordinaten van twee posities gelijk zijn, en False als dat niet zo is. Methode __repr__ is de methode die een object van die klasse als string weergeeft. We overschrijven ook die methode zodat de cordinaat wordt weergegeven.

Stel nu dat we de positie op het vakje in rij 1 en kolom 2 willen opslaan, dan instantiren we een Position object met cordinaten irow=1 en icol=2. We kennen het object toe aan variabele p1:

Pad¶

We moeten ook het reeds afgelegde pad kunnen bijhouden, en dus definiren we een klasse Path. Dat pad bestaat uit een reeks van opeenvolgende posities, die we meegeven met de constructor, die op zijn beurt de reeks toekent aan attribuut positions. We definiren ook een property length die de padlengte geeft, en properties irow en icol die de cordinaten als NumPy array geven. Een NumPy array is net als een list, maar bedoeld voor numerieke elementen, waarop je dan allerhande matrixbewerkingen kan uitvoeren. We voorzien ook nog een methode contains die checkt of een gegeven positie al dan niet tot het pad behoort, en een methode add die een nieuwe positie aan het pad toevoegt en een nieuw Path object retourneert met het nieuwe pad. Tot slot overschrijven we opnieuw de __repr__ methode om het pad mooi weer te geven als string.

In [202]:
import numpy as np

class Path:

    def __init__(self, *positions):
        self.positions = positions
    
    @property
    def length(self):
        return len(self.positions)
    
    @property
    def irow(self):
        return np.array([position.irow for position in self.positions])

    @property
    def icol(self):
        return np.array([position.icol for position in self.positions])

    def contains(self, position):
        return position in self.positions
    
    def add(self, position):
        return Path(*self.positions, position)

    def __repr__(self):
        positions = ", ".join([str(position) for position in self.positions])
        return f"[{positions}]"

Doolhof¶

Vervolgens moeten we ook de doolhofconfiguratie implementeren. We definiren hiervoor een klasse Maze die een doolhof als een vierkant rooster voorstelt met n x n vakken. Het aantal vakken n wordt in attribuut size opgeslagen. Het rooster wordt gedefinieerd a.d.h.v. een 2D NumPy array, die in attribuut grid wordt opgeslagen, en als elementen integers bevat met de volgende code:

  • 0: startvakje
  • 1: vrij vakje
  • 2: vakje met barrire
  • 3: eindvakje

De grid array wordt aan de constructor meegegeven. Om ons het leven wat makkelijker te maken, definiren we een statische methode create die een maze list van strings bevat met de volgende symbolen:

  • '*': startvakje
  • '.': vrij vakje
  • '#': vakje met barrire
  • 'o': eindvakje

De methode zet de list van strings om naar een grid array, instantieert een nieuw Maze object, en retourneert dit. De __repr__ methode die we opnieuw overschrijven, retourneert trouwens een string die de doolhofconfiguratie met dezelfde symbolen.

We definiren verder nog methoden get_start en get_goal die de start- en eindposities retourneren als Position objecten. De methode is_valid checkt of een gegeven positie al dan niet geldig is. Een positie is niet geldig als het buiten het rooster valt of naar een vakje verwijst met een barrire. Tenslotte is er nog een plot methode waarmee we het doolhof kunnen plotten. Hiervoor maken we gebruik van de MatPlotLib library.

In [208]:
import matplotlib.pyplot as plt
from matplotlib import colors

class Maze:
   
    symbols = ('*', '.', '#', 'o')
    
    def __init__(self, grid):
        # grid is integer array:
        # start = 0, free = 1, wall = 2, goal = 3
        self.grid = np.array(grid)
        self.size = self.grid.shape[0]

    def get_start(self):
        irow, icol = np.where(self.grid == 0)
        return Position(irow[0], icol[0])

    def get_goal(self):
        irow, icol = np.where(self.grid == 3)
        return Position(irow[0], icol[0])
    
    def is_valid(self, position):
        # checkt of een positie geldig is: moet binnen het grid liggen en mag geen barrire zijn
        return (0 <= position.irow < self.size and
                0 <= position.icol < self.size and 
                self.grid[position.irow, position.icol] != 2)
    
    def plot(self):
        # colormap
        cmap = colors.ListedColormap(['forestgreen', 'lightyellow', 'purple', 'red'])
        bounds = np.linspace(-0.5, 3.5, 5)
        norm = colors.BoundaryNorm(bounds, cmap.N)
        # plot grid
        plt.matshow(self.grid, cmap=cmap, norm=norm)
        # axes
        xmax = self.size - 0.5
        ymax = self.size - 0.5
        ax = plt.gca()
        ax.set_aspect("equal")
        ax.set_xticks(np.linspace(-0.5, xmax, self.size + 1))
        ax.set_xticklabels([])
        ax.set_yticks(np.linspace(-0.5, ymax, self.size + 1))
        ax.set_yticklabels([])
        ax.tick_params(axis=u'both', which=u'both', length=0)
        # plot grid and set axis limits
        plt.grid()
        plt.xlim([-0.5, xmax])
        plt.ylim([ymax, -0.5])
        
    def __repr__(self):
        return "\n".join([''.join([self.symbols[item] for item in row]) for row in self.grid])

    @staticmethod
    def create(maze):
        # creert grid adhv list van strings "maze"
        return Maze(grid=[[Maze.symbols.index(symbol) for symbol in row] for row in maze])

State-space OO-model¶

Voor we verder gaan implementeren we eerst een OO-model waarin de volgende concepten een plaats krijgen:

  • state: een klasse waarin je de toestand van het zoekprobleem kan weergeven. Op een toestand moet je een move kunnen toepassen om naar een andere toestand te gaan.
  • goal: van een bepaalde state moet je kunnen evalueren of die in de goaltoestand is of niet.
  • move: een move bepaalt hoe je van de ene naar de andere toestand kan overgaan

Deze OO-structuur biedt een generiek kader waarin we de zoekproblemen kunnen formuleren. Na het implementeren van dit generieke kader, wordt een specifieke implementatie voor het doolhofprobleem opgesteld.

We starten met de abstracte klasse State. Voor een bepaalde state moeten we de mogelijke moves kunnen berekenen, kunnen bepalen of het al dan niet om de eindtoestand gaat, en een move kunnen toepassen. Voor elk van die 3 acties definiren we een abstracte methode die dan voor het oplossen van een specifiek probleem via overerving zal moeten gemplementeerd worden.

In [213]:
from abc import ABC, abstractmethod

class State(ABC):

    @abstractmethod
    def calculate_moves(self):
        # Op een state wordt deze methode aangeroepen om volgens de production rules
        # de moves te berekenen die het probleem in een nieuwe toestand kunnen brengen.
        # Deze functie geeft dus een lijst van moves terug (eventueel leeg).
        pass

    @abstractmethod
    def apply_move(self, move):
        # Op een state kan een move toegepast worden om naar een volgende state over te gaan.
        # Deze functie geeft dus een nieuwe state terug.
        pass
        
    @abstractmethod
    def is_goal(self):
        # Controleert of een state de `goal state` is.
        # Retourneert dus een booleaanse waarde.
        pass

En dan hebben we nog de klasse Move, die een State object bevat, en een zichzelf zal toepassen op die toestand.

In [214]:
class Move:
    
    def __init__(self, state):
        self.state = state
    
    def apply(self):
        return self.state.apply_move(self)

Maze state-space representation¶

We maken nu een klasse MazeMove aan, op basis van de klasse Move. We passen m.a.w. overerving toe, waarbij MazeMove een subklasse is van Move.

Uiteraard moet er in de constructor ook probleemspecifieke informatie meegegeven worden. Hier moeten we met die informatie naar een volgende positie kunnen gaan. We doen dit door drow en dcol te definiren. De formules zijn:

irow_new = irow + drow

icol_new = icol + dcol

Met methode new_position wordt die nieuwe positie op die manier berekent en geretourneerd als nieuw Position object. Let wel dat enkel de nieuwe positie wordt berekend, maar de move wordt nog niet toegepast. M.a.w. de toestand wordt nog niet gewijzigd, want dat is de taak van de overgerfde methode apply.

Merk ook op dat we hier niet redeneren vanuit de generieke klasse State, maar vanuit het specifieke doolhofprobleem dat we gaan oplossen.

We overschrijven ook methode __repr__ om een stringrepresentatie te laten teruggeven van de move. De productieregels schrijven voor dat we enkel naar links, rechts, boven of onder kunnen gaan. De strings zijn daarom 'left, 'right, 'up', 'down'.

In [215]:
class MazeMove(Move):

    def __init__(self, state, drow=0, dcol=0):
        super().__init__(state)
        self.drow = drow
        self.dcol = dcol
    
    def new_position(self):
        return Position(self.state.position.irow + self.drow, self.state.position.icol + self.dcol)

    def __repr__(self):
        if self.dcol == -1:
            return 'left'
        if self.dcol == 1:
            return 'right'
        if self.drow == -1:
            return 'up'
        if self.drow == 1:
            return 'down'

Omdat we niet telkens zouden moeten nadenken welke waarden voor drow en dcol we moeten meegeven, definiren we klassen Left, Right, Up en Down, die uiteraard overerven van superklasse MazeMove:

In [216]:
class Left(MazeMove):    
    def __init__(self, state):
        super().__init__(state, dcol=-1)

class Right(MazeMove):
    def __init__(self, state):
        super().__init__(state, dcol=1)

class Up(MazeMove):
    def __init__(self, state):
        super().__init__(state, drow=-1)

class Down(MazeMove):
    def __init__(self, state):
        super().__init__(state, drow=1)

Tenslotte moeten we nog de toestand implementeren a.d.h.v. klasse MazeState die uiteraard overerft van abstracte superklasse State. Herinner je dat een toestand wordt gekenmerkt door een doolhofconfiguratie, een huidige positie en een reeds afgelegd pad. Vandaar dat de constructor een Maze object, een Position object en een Path object accepteert, die in attributen maze, position en path worden opgeslagen. Bemerk dat de huidige positie geen deel uitmaakt van ons reeds afgelegd pad, vandaar dat we een property full_path definiren die de huidige positie wel bevat.

Uiteraard moeten we dan de abstracte methodes implementeren: calculate_moves geeft ons alle geldige moves. Vandaar dat we ook een methode is_valid definiren die checkt of een nieuwe positie geldig is binnen het doolhof n niet tot het reeds afgelegd pad behoort, want we willen niet op ons stappen terugkeren. Met methode apply_move kunnen we dan n van die moves gaan toepassen waardoor we in een nieuwe toestand komen. Die methode retourneert dan ook een nieuw MazeState object die deze nieuwe toestand vastlegt. Tenslotte checkt is_goal of de toestand de eindtoestand is.

Met de plot methode wordt het doolhof uitgeprint en het afgelegde pad en huidige positie worden toegevoegd aan de plot. De dynamic_plot visualiseert het afgelegde pad dynamisch, waarbij inputargument wait bepaalt hoeveel seconden - standaard een halve seconde - er moet gewacht worden om de volgende plot te tonen. Tenslotte wordt ook de __repr__ overschreven om het doolhof weer te geven samen met het pad.

In [217]:
from IPython.display import clear_output
from time import sleep


class MazeState(State):

    def __init__(self, maze, position, path):
        self.maze = maze
        self.position = position
        self.path = path
    
    @property
    def full_path(self):
        return self.path.add(self.position)

    def calculate_moves(self):
        if self.is_goal():
            return []  # geen moves meer als we in de eindtoestand zijn
        moves = [Left(self), Right(self), Up(self), Down(self)]
        return [move for move in moves if self.is_valid(move)]

    def is_valid(self, move):
        new_position = move.new_position()
        return self.maze.is_valid(new_position) and not self.path.contains(new_position)

    def apply_move(self, move):
        return MazeState(self.maze, move.new_position(), self.full_path)  # nieuwe toestand

    def is_goal(self):
        return self.maze.get_goal() == self.position

    def plot(self):
        self.maze.plot()
        x = np.arange(0, self.maze.grid.size)
        plt.plot(x[self.position.icol], x[self.position.irow], 'bo')
        if self.path.length > 0:
            plt.plot(x[self.path.icol], x[self.path.irow], 'ko', )

    def dynamic_plot(self, wait=0.5):
        x = np.arange(0, self.maze.grid.size)
        icol, irow = [], []
        for position in self.path.positions:
            self.maze.plot()
            irow.append(position.irow)
            icol.append(position.icol)
            plt.plot(x[icol], x[irow], 'ko')
            plt.show()
            sleep(wait)
            clear_output()
        self.plot()

    def __repr__(self):
        full_path = self.full_path
        if full_path.length > 1:
            symbols = Maze.symbols + ('x', '!')
            grid = self.maze.grid.copy()
            full_path = self.full_path
            irow, icol = full_path.irow[1:], full_path.icol[1:]
            if grid[irow[-1], icol[-1]] == 3:
                grid[irow[:-1], icol[:-1]] = 4
                grid[irow[-1], icol[-1]] = 5
            else:
                grid[irow, icol] = 4
            return "\n".join([''.join([symbols[item] for item in row]) for row in grid])
        else:
            return str(self.maze)

import numpy as np

class Path:

    def __init__(self, *positions):
        self.positions = positions
    
    @property
    def length(self):
        return len(self.positions)
    
    @property
    def numbers(self):
        return np.array([position.number for position in self.positions])

    def contains(self, position):
        return position in self.positions
    
    def add(self, position):
        return Path(*self.positions, position)

    def __repr__(self):
        positions = ", ".join([str(position) for position in self.positions])
        return f"[{positions}]"

# Tenslotte definiren we ons bord met putten die een pion terug naar start sturen

class GooseGameBoard:

    def __init__(self, pitfalls=[], start=1, goal=63):
        self.start = start
        self.goal = goal
        self.pitfalls = pitfalls

    def get_start(self):
        return Position(self.start)
        
    def get_goal(self):
        return Position(self.goal)
    
    def is_valid(self, position):
        # checkt of een positie geldig is: moet binnen de range van start en goal liggen
        return self.start <= position.number <= self.goal
    
    def to_dict(self):
        # omzetten naar dictionary kan handig zijn
        # een Python dictionary is equivalent met een Java HashMap
        return dict(start=self.start, goal=self.goal, putten=self.pitfalls)
    
    def __repr__(self):
        return str(self.to_dict())

# klasse GooseGameState moet erven van State

class GooseGameState(State):

    def __init__(self, board, position, path):
        self.board = board
        self.position = position
        self.path = path
    
    @property
    def full_path(self):
        return self.path.add(self.position)

    def calculate_moves(self):
        if self.is_goal():
            return []  # geen moves meer als we in de eindtoestand zijn
        moves = [Een(self), Twee(self), Drie(self), Vier(self), Vijf(self), Zes(self)]
        return [move for move in moves if self.is_valid(move)]

    def is_valid(self, move):
        return self.board.is_valid(move.new_position())

    def new_position(self, move):
        new_position = move.new_position()
        if new_position.number in self.board.pitfalls:
            return self.board.get_start()  # terug naar start bij put
        if new_position.number > self.board.goal: 
            return self.position  # blijven staan als je op 't einde te veel gooit
        return new_position

    def apply_move(self, move):
        return GooseGameState(self.board, self.new_position(move), self.full_path)  # nieuwe toestand

    def is_goal(self):
        return self.board.get_goal() == self.position

    def to_dict(self):
        state = self.board.to_dict()
        state["vakje"] = self.position
        state["vorige"] = self.path
        return state

    def __repr__(self):
        return str(self.to_dict())
